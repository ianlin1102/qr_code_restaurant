## Summary of Recurring Issues

### Critical (affects usability on mobile)

1. **Small touch targets (< 44px)** — Found in 18+ components. Common culprits:
   - Quantity +/- buttons at `h-8 w-8` (32px) or `w-6 h-6` (24px): `OrderingSheet`, `ItemCustomizeView`, `CreateSplitSheet`, `SettlementSheet`, `MenuManagePage` sidebar
   - Inline action buttons/links with only `text-xs` and no min-h: `BillSettleDialog` remove coupon, `MenuItemForm` discount bar, `TablesPage` void button, `MenuPage` set name link
   - Close/delete micro-buttons: `CsvImportDialog` close, `FloorPlanEditorPage` zone delete, `MenuManagePage` cart remove

2. **`text-[10px]` usage** — Found in 10+ components. At 10px, this is below the minimum readable size on most mobile devices. Used for: option tags, status pills, staff badges, paid indicators.

### Moderate

3. **Inconsistent input height** — Base `Input` is `h-9` (36px). Some consumers add `min-h-[44px]`, others don't. No consistent approach.

4. **Native HTML elements mixed with ui components** — `<select>`, `<input type="date">`, raw `<button>`, `<input type="number">` used alongside shadcn components in: `TableCrudDialog`, `AnalyticsPage`, `CsvImportDialog`, `TipSelector`, `MenuItemForm` discount bar.

5. **Missing cursor-pointer** — Clickable elements using `onClick` on `<span>`, `<div>`, or `<button>` without explicit `cursor-pointer`: `MenuPage` customer name, `MenuItemTable` InlineEdit (has it), `CategoryManagePage` sort order.

### Minor

6. **No sm:/md: responsive prefixes in some components** — Several components are mobile-first by default or use Dialog/Sheet overlays, so breakpoints aren't always needed. However `ActiveOrdersSidebar` is fixed at `w-80` with no responsive adjustment.

7. **Form double-submit protection** — Generally well-implemented. All identified forms have `disabled` on submit buttons during async operations.

8. **Loading/Empty states** — Generally well-implemented. All API-fetching components have loading indicators. List components have empty states. The only gap is `LangSelectPage` which doesn't show loading while fetching store info.
