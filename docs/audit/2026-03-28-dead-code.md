## Recommendations

### Immediate (High Priority)
1. **Fix or remove `SplitBillDialog`**: Either delete the deprecated component and update `TablesPage.tsx` + `TableDetailPanel.tsx` to use `BillSettleDialog` exclusively, or restore `api.createSplitBill` + backend route. Currently it crashes at runtime.

### Short Term (Low Priority)
2. **Remove unused exports**: `createSplitPaymentIntent`, `getStaffPerformance`, `getTableById`, `updateTableStatus`, `getRoleById`, `createUser` -- all defined but never called. Removing them reduces code surface and confusion.
3. **Remove `api.getActiveBill`**: Not called anywhere. If needed later, it can be re-added.
4. **Remove `usePermission.ts`**: Never imported. If client-side permission checks are needed later, it can be rebuilt.
5. **Remove `StoreUser` type**: Zero consumers. The `AuthUser` type is what the codebase actually uses.

### Decide (Keep or Remove)
6. **`POST /tables/:tableId/settle` route**: Superseded by `POST /bills/:billId/settle`? If so, remove the route and `settleTable` function.
7. **`PATCH /waitlist/:entryId` route**: No frontend UI exists for editing waitlist entries. Keep if planned, remove if not.
8. **`GET /auth/me` route**: Useful for session validation on page refresh. Consider wiring it up or removing.
9. **Printer config routes**: Keep if printer management UI is planned. Remove if printers are managed externally.
10. **`seed-features.ts`**: Register in `package.json` scripts or document as manual-run utility.
