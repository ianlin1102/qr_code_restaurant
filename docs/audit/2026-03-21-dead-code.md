## 5. Recommended Actions

### Immediate (safe to remove)
1. **Delete `MenuItemEditSheet.tsx`** — `MenuItemForm` already handles create/edit functionality and is actively used by `MenuManagePage`
2. **Remove dead types** from `shared/types.ts`: `StoreUser`, `Role`, `LoginRequest`, `UpdateOrderStatusRequest`, `UpdateOrderItemsRequest` — these add confusion without being consumed
3. **Remove `api.settleTable`** from `api.ts` — no caller exists. If settle functionality is needed in the future, it can be re-added
4. **Remove `api.updateWaitlistEntry`** from `api.ts` — the waitlist UI only uses add/seat/remove

### Requires Decision
5. **`GET /auth/me`** — either build frontend session-restore to use it (check token on page load) or remove the endpoint
6. **Printer config routes** — either build the frontend printer settings UI or remove the routes. The `reprintOrder` route IS used, but config management is not
7. **`GET /health`** — keep if used for deployment health checks or monitoring; remove if not

### Types to Reconsider
8. **`StoreUser`** — once Prisma migration is done, this type may become relevant again. Consider keeping but marking with a `@todo` comment
9. **`LoginRequest`** / `UpdateOrderStatusRequest` / `UpdateOrderItemsRequest` — these are well-intentioned request types that routes should validate against. Consider actually using them in route handlers instead of removing them
