## Recommendations

1. **Keep but wire up**: `GET /auth/me`, `POST /settle`, printer config routes -- these represent planned features. Add UI or remove the routes.
2. **Remove**: `StoreUser`, `LoginRequest`, `UpdateOrderStatusRequest`, `UpdateOrderItemsRequest` -- type definitions with zero consumers. If needed later, they can be re-added.
3. **Clean up**: Remove the `verifyToken` import from `auth.routes.ts`. Remove the `export` from `formatPrice` (or keep it if you plan external use).
4. **Decide**: `api.settleTable` and `api.updateWaitlistEntry` -- either build the UI that uses them or remove the dead wrappers.
5. **Register or remove**: `seed-features.ts` -- add a `package.json` script entry or delete the file.
