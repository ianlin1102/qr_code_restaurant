### Structural anomaly: duplicate key in admin.json

Both zh and en `admin.json` have a duplicate key `"staff"` inside `nav`:
```json
"nav": {
  "staff": "员工管理",   // line 11
  "staff": "员工",       // line 15  <-- overwrites line 11
}
```
The second value overwrites the first. This means `nav.staff` resolves to `"员工"` (zh) / `"Staff"` (en), and the value `"员工管理"` / `"Staff"` on line 11 is effectively dead code.

## Priority Summary

### P0 -- Critical (Chinese users see English / English users see Chinese)

1. **`staff.*` i18n keys completely missing from both zh and en JSON** -- entire Staff Management page falls back to English `defaultValue` strings. Chinese users see all-English UI.
2. **`login.passwordPlaceholder` missing from both JSON files** -- minor, falls back to English.
3. **Backend `table.service.ts` error messages hardcoded in Chinese** -- English-speaking admins see Chinese error messages when creating duplicate tables or deleting occupied tables.
4. **`StaffManagePage` role labels `Owner`/`Staff` hardcoded in English** (lines 39-40, 155) -- Chinese users see English role names.

### P1 -- Medium (bypasses i18next, maintenance burden)

5. **~25 instances of `lang === 'zh' ? ... : ...` pattern** across customer pages (MenuPage, CheckoutPage, CartPage, OrderConfirmPage, OrderHistoryPage, LoginPage, etc.) -- functionally works but bypasses i18next, making translations harder to maintain and export.
6. **`MenuItemDetailSheet` quick tags** use hardcoded arrays instead of i18next.
7. **`qr-pdf.ts`** "Scan to Order" and pop-up alert are English-only.
8. **`admin.json` duplicate `nav.staff` key** -- one value is silently discarded.

### P2 -- Low (data model gaps, future-proofing)

9. **`Store` missing `announcementEn`** -- customer-visible announcements cannot be bilingual.
10. **`CartItem` missing `nameEn`** -- cart displays only `name` field.
11. **`Order.tableName` missing `tableNameEn`** -- order snapshots only store one language.
12. **`store.service.ts` `updateStore()` ignores `nameEn`/`descriptionEn`**.
13. **`menu.service.ts` `getMenu()` response does not include `store.nameEn`/`descriptionEn`** in the `Pick<Store, ...>`.
14. **`LangSelectPage` English-only strings** (`Experience personalized...`, `International Standard`) -- shown before language is selected, acceptable but inconsistent.
15. **`MenuPage` announcement dialog badge** "New Release" is English-only.
