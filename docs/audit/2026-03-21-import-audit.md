## Observations (Non-Blocking)

### Mixed Import Style Between Client and Server

The client and server use different import extension conventions:
- **Client**: No file extensions (Vite bundler mode)
- **Server**: Explicit `.js` extensions (ESM convention for `tsx` runtime)

This is technically correct for both runtimes, but introduces a stylistic inconsistency within the monorepo. Not a bug, just a convention difference worth documenting.

### Unused Shared Type Exports

5 types in `shared/types.ts` are exported but never imported:
- `LoginRequest` -- could be used if login form validation is added server-side
- `Role` -- used inline as `'owner' | 'staff'` in some places instead
- `StoreUser` -- the full user model; code uses `AuthUser` (subset) instead
- `UpdateOrderItemsRequest` -- defined but API uses `OrderItem[]` directly
- `UpdateOrderStatusRequest` -- defined but API uses `{ status: OrderStatus }` directly

These are not broken imports, but could be cleaned up if unused long-term.
