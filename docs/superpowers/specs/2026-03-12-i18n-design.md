# i18n Design Spec

## Goal

Add full internationalization (Chinese/English) to the QR ordering system — both UI text and menu item data.

## Data Model Changes

Add optional English fields alongside existing fields. `name` remains required as the primary/fallback name.

### shared/types.ts

```typescript
// MenuItem — add:
nameEn?: string
descriptionEn?: string

// Category — add:
nameEn?: string

// MenuItemOption — add:
nameEn?: string

// MenuItemOptionChoice — add:
nameEn?: string
```

Display logic: when `lang=en`, prefer `nameEn`; fall back to `name` if absent.

### Seed Data

Add English names to existing seed data for both stores (store-demo-001, store-demo-002).

## i18n Library

react-i18next with namespace separation.

### File Structure

```
client/src/i18n/
├── index.ts          # i18next init + config
├── zh/
│   ├── common.json   # shared: buttons, status labels
│   ├── customer.json # customer-facing pages
│   └── admin.json    # admin dashboard pages
└── en/
    ├── common.json
    ├── customer.json
    └── admin.json
```

### Language Detection Priority

1. URL param `?lang=en` (highest)
2. localStorage key `i18n-lang`
3. `navigator.language`
4. Default: `zh`

## Customer Flow

1. Scan QR → check localStorage for `i18n-lang`
   - Exists: skip to menu page
   - Missing: show language selection page
2. Language selection page: two buttons (中文 / English)
   - On select: save to localStorage, navigate to menu
3. Menu page header: small toggle to switch language anytime

### New Route

`/lang-select/:storeId/:tableId` — language selection page, redirects to `/scan/:storeId/:tableId` after selection.

## Admin Flow

### Collapsible Sidebar

- Expanded: `w-56`, icon + text (current behavior)
- Collapsed: `w-16`, icons only
- Toggle button in header area
- Collapse state persisted in localStorage

### Language Toggle

- Bottom of sidebar: globe icon button
- Expanded: shows `🌐 中文/EN`
- Collapsed: shows `🌐` only
- Switches all admin UI text between zh/en

### Menu Management

- MenuManagePage: add `nameEn` / `descriptionEn` input fields for each menu item
- Category management: add `nameEn` field
- Option/choice editing: add `nameEn` fields

## Display Helper

Utility function for bilingual field resolution:

```typescript
// lib/i18n-utils.ts
export function localized(item: { name: string; nameEn?: string }, lang: string): string {
  return lang === 'en' ? (item.nameEn || item.name) : item.name
}
```

## Out of Scope

- Additional languages beyond zh/en
- Server-side i18n (API error messages stay in English)
- RTL support
