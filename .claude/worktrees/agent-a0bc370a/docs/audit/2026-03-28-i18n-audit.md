# i18n Audit Report -- 2026-03-28

## Executive Summary

The project has a **dual admin translation system** problem: `admin.ts` (used by most admin components via `useT()`) and `admin.json` (used by 2 components via `useTranslation('admin')`) have **diverged significantly** with different key naming conventions and 299+ keys only in one system. Customer-side translations are in good shape with only minor issues.

**Severity breakdown:**
- Critical: 1 (dual admin i18n system desync)
- High: 8 (hardcoded user-visible strings bypassing i18n)
- Medium: 7 (missing keys, swapped values, hardcoded locale)
- Low: 5 (minor inconsistencies)

## Layer 2: Admin Inline Translations (admin.ts)

### admin.ts zh vs en

**Keys in sync**: All keys match perfectly between zh and en branches in admin.ts. No missing translations in either direction.

### CRITICAL: admin.ts vs admin.json Divergence

The project has **two separate admin translation systems** that have diverged:

1. **`admin.ts`** (inline object, used by `useT()`) -- 35+ admin components
2. **`admin.json`** (i18next JSON, used by `useTranslation('admin')`) -- 2 components only: `OrderReceipt.tsx`, `OrderEditMode.tsx`

**Divergence stats:**
- 299 keys exist in admin.ts but NOT in admin.json
- 107 keys exist in admin.json but NOT in admin.ts

**Key naming convention conflicts** (same data, different namespace paths):

| admin.ts path | admin.json path |
|---------------|-----------------|
| `categories.*` | `categoryManage.*` |
| `settings.*` | `storeSettings.*` |
| `coupons.types.*` | `coupons.percentage` / `coupons.fixedAmount` / `coupons.bogo` |
| `tables.status.*` | `tables.idle` / `tables.occupied` (flat) |

**Keys only in admin.json (not in admin.ts)** -- these 107 keys represent features that were added to admin.json but not synced to admin.ts, or vice versa. Notable groups:
- `categoryManage.*` (18 keys) -- entire namespace unique to JSON
- `storeSettings.*` (18 keys) -- entire namespace unique to JSON
- `tables.summary`, `tables.printAll`, `tables.scanToOrder`, `tables.print`, `tables.settle` etc.
- `receipt.title`, `receipt.customer`, `receipt.time`, `receipt.thanks`
- `tableDetail.orderNumber`, `tableDetail.items`, `tableDetail.addItem`, `tableDetail.quickDiscount`, `tableDetail.free`, `tableDetail.subtotal`, `tableDetail.discountLabel`, `tableDetail.save`, `tableDetail.cancel`, `tableDetail.noItems`, `tableDetail.noResults`

**Keys only in admin.ts (not in admin.json)** -- 299 keys including entire namespaces:
- `bill.*` (18 keys)
- `splitBill.*` (13 keys)
- `transferTable.*` (5 keys)
- `staff.*` (16 keys)
- `csv.*` (16 keys)
- `roles.*` (17 keys including `permLabels.*`)
- `settings.*` (23 keys -- admin.ts version)
- `categories.*` (18 keys -- admin.ts version)
- All `common.*` keys (30 keys)
- All `dashboard.status.*` keys (5 keys)

## Priority Summary

### P0 -- Architecture Decision Required

1. **Unify admin i18n system**: Choose ONE system (recommend keeping `admin.ts` via `useT()` since 35+ components use it) and migrate the 2 components using `useTranslation('admin')` (OrderReceipt, OrderEditMode). Then either delete `admin.json` or auto-generate it from `admin.ts` for any i18next-dependent tooling.

### P1 -- Missing Translations (user-visible bugs)

2. **Add `menu.emptyCategory` key** to both zh and en `customer.json`
3. **Fix inline `lang ===` ternaries** in OrderConfirmPage (13 strings), MenuPage (6 strings), CartPage (2 strings), OrderHistoryPage (2 strings), CheckoutPage (2 strings) -- extract to customer.json keys
4. **Fix hardcoded English in admin components**: `StaffManagePage.tsx` ("No roles", "perms"), `AnalyticsPage.tsx` ("No data" in RevenueChart), `OrderEditMode.tsx` ("Note..." placeholder)

### P2 -- Consistency / Maintenance

5. **Fix hardcoded `'zh-CN'` locale** in OrderReceipt and OrderDetailDialog date formatting
6. **Fix server-side Chinese error** in `table.service.ts` line 140
7. **Remove `|| 'fallback'` patterns** in BillSettleDialog, CsvImportDialog, StoreSettingsPage -- these keys all exist in admin.ts, the fallbacks are unnecessary noise
8. **Add `login.usernamePlaceholder`** to admin.ts or use existing key

### P3 -- Low Priority / Cosmetic

9. LangSelectPage hardcoded strings are intentional for the bilingual selection UI
10. `LoginPage.tsx` `placeholder="admin"` -- minor, example text
11. `CsvImportDialog.tsx` "Row N: reason" -- only shown during CSV import errors
